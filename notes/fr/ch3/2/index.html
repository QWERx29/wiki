    3.2 线段树 I

    3.2.1 线段树

    线段树是一种基于分治思想，快速处理区间问题的数据结构。

    线段树的核心在于将每个区间拆分成不超过 \(2 \log n\) 个线段树上的区间。

    因此，只要线段树上的区间可以快速修改且可以快速合并，就可以用线段树解决。

    线段树可以解决树状数组所能解决的一切问题，但常数约为树状数组的2-4倍，而且代码量远高于树状数组。

    下面给出线段树实现[FAIOJ104]的部分代码

    #define lc (x<<1)
#define rc (x<<1|1)
#define mid ((l+r)>>1)
int a[N];
ll s[N<<2];
inline void pu(int u){
    s[x]=s[lc]+s[rc];
}
void bld(int x,int l,int r){
    if(l==r){s[x]=a[l];return;}
    bld(lc,l,mid),bld(rc,mid+1,r);
    pu(x);
}
void upd(int x,int l,int r,int pos,int v){
    if(l==r){s[x]+=v;return;}
    if(pos<=mid)upd(lc,l,mid,pos,v);
    else upd(rc,mid+1,r,pos,v);
    pu(x);
}
ll qry(int x,int l,int r,int L,int R){
    if(l==L&&r==R)return s[x];
    if(R<=mid)return qry(lc,l,mid,L,R);
    if(L>mid)return qry(rc,mid+1,r,L,R);
    return qry(lc,l,mid,l,mid)+qry(rc,mid+1,r,mid+1,R);
}

    3.2.2 线段树的懒惰标记

    懒惰标记是线段树的另一个核心思想。

    当我们对线段树上的一整个区间进行修改时，它的子区间也要修改。然而子区间的数量是 \(O(n)\) 级别的，如果每次都全部修改，复杂度将会退化到 \(O(n^2)\) 。

    因此我们在修改一个区间时，先把修改作为一个标记放在这个线段树节点上，等到下次遍历到这个节点的时候再下放标记。

    下面给出线段树实现[FAIOJ106]的代码。

    #define lc (x<<1)
#define rc (x<<1|1)
#define mid ((l+r)>>1)
int a[N];
ll s[N<<2],tag[N<<2];
inline void pu(int u){
    s[x]=s[lc]+s[rc];
}
void bld(int x,int l,int r){
    if(l==r){s[x]=a[l];return;}
    bld(lc,l,mid),bld(rc,mid+1,r);
    pu(x);
}
inline void add(int x,int l,int r,ll v){
    s[x]+=(r-l+1)*v,tag[x]+=v;
}
inline void pd(int x,int l,int r){
    if(tag[x])add(lc,l,mid,tag[x]),add(rc,mid+1,r,tag[x]),tag[x]=0;
}
void upd(int x,int l,int r,int L,int R,ll v){
    if(l==L&&r==R){add(x,l,r,v);return;}
    pd(x,l,r);
    if(R<=mid)upd(lc,l,mid,L,R,v);
    else if(L>mid)upd(rc,mid+1,r,L,R,v);
    else upd(lc,l,mid,L,mid,v),upd(rc,mid+1,r,mid+1,R,v);
    pu(x);
}
ll qry(int x,int l,int r,int L,int R){
    if(l==L&&r==R)return s[x];
    pd(x,l,r);
    if(R<=mid)return qry(lc,l,mid,L,R);
    if(L>mid)return qry(rc,mid+1,r,L,R);
    return qry(lc,l,mid,L,mid)+qry(rc,mid+1,r,mid+1,R);
}

    例题3.2.4：[FAIOJ30129]维护序列（10）

    本题要求支持区间加、区间乘、区间求和

    对于区间加和区间乘这两个操作，显然需要分别打标记。

    但这样就引出了新的问题：标记之间的影响如何处理？

    当某个节点区间要整体加 \(x\) 时，如果这个节点已有乘 \(y\) 的标记，那么这个乘法标记会被“破坏”，因此要先把它下放下去。

    3.2.3 线段树维护区间信息

    线段树处理维护区间和以外，还可以维护很多区间信息。只要该区间信息支持快速合并即可由线段树快速维护。

    例题3.2.7 [FAIOJ3551]线性函数（10）

    本题维护的不是整数序列，而是函数序列。

    函数的复合运算满足结合律。所以可以用线段树。考虑线性函数的复合运算如何合并。

    若 \([l,mid]\) 复合形成的函数是 \(k_1x+b_1\) ， \([mid+1,r]\) 复合形成的函数是 \(k_2x+b_2\) ，则 \([l,r]\) 复合形成的函数是 \(k_2(k_1x+b_1)+b_2=k_1k_2x+(k_2b_1+b_2)\)

    因此对于线段树每个区间维护 \(k,b\) 即可。

    查询时不要直接在线段树的节点中代入 \(x\) ，而是把整个区间的 \(kx+b\) 算出后再代入。

    例题3.2.8 [FAIOJ3050]区间最大子段和（11）

    本题要求支持单点修改，区间最大子段和。

    考虑最大子段和如何合并，即如何由 \([l,mid]\) 和 \([mid+1,r]\) 的某些信息得到 \([l,r]\) 的最大子段和？

    大区间的最大子段和可以三种情况：完全包含于左区间；完全包含于右区间；跨左右区间。

    前两种情况分别是 \(ans_L\) 和 \(ans_R\) ；

    第三种情况比较复杂，因为左右区间的最大子段和并不一定会取得。

    但在这种情况下，左区间一定取得一个后缀，而右区间一定取得一个前缀。

    因此若答案最大，那么左区间必取最大后缀，而右区间必取最大前缀。

    所以对于每个区间节点，同时维护 \(sum,pre\) 和 \(suf\) 表示和、最大前缀和最大后缀即可。

    注意！下面的内容将大幅减少你的代码难度！

    因此本题需要维护三个信息。而这三个信息在我们的qry函数中同样是需要的。

    一个函数无法同时返回三个值，所以需要用结构体来同时存储这三个值，并重载运算符+来支持合并。

    这样做的好处在于你几乎不用修改upd和qry的代码，所有操作都在重载的+中完成了。

    struct node{
    int sum,pre,suf,ans;
    node(){sum=0,pre=suf=ans=-inf;}
    node operator+(const node& p)const{
        node res;
        res.sum=sum+p.sum;
        res.pre=max(pre,sum+p.pre);
        res.suf=max(p.suf,p.sum+suf);
        res.ans=max(max(ans,p.ans),suf+p.pre);
        return res;
    }

    例题3.2.9 [FAIOJ21002][SDOI2010]序列操作（12）

    本题要求支持 \(4\) 种操作：区间覆盖，区间取反，区间求和，求区间极长连续段1。

    前三种操作都是前面的线段树基本操作。覆盖和取反标记的影响很容易处理：

    覆盖消灭取反标记；取反反转覆盖标记。

    问题在于最后一种操作。

    对于一个区间，其最长连续段的位置分为三种情况：

    完全在左区间；完全在右区间；跨两区间。

    如果完全在一个区间，那么答案就是max(ans[lc],ans[rc])。

    如果跨区间呢？？

    那么它只能是左区间最右面的几个1和右区间最左面的几个1。因此维护pre和suf分别表示一个区间最左面和最右面1的个数即可。

    然而仅仅维护1的这些信息是不够的。由于取反会让0变1，1变0，所以还要维护关于0的pre、suf和ans。

    因此对于每个线段树上的节点，需要维护 \(7\) 个信息和 \(2\) 个标记。本题非常考验代码能力！！！

    同样地，使用结构体打包这 \(7\) 个信息，并重载+运算符。在询问区间极长连续时直接返回整个结构体即可。
